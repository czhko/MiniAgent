MiniAgent — 基于状态机的多模型分工框架

核心矛盾：小说生成涉及世界观、人设、人物关系、元事件、事件链、规则、文风、输出格式八个维度。Transformer 注意力机制随上下文增长而消散——短对话尚可，长线必然崩溃。模型提供下限，框架决定上限。如同电影需要编剧、演员、导演、剪辑、审核各司其职，指望单一模型同时承担意图分析、状态管理、剧情生成、质量校验等多重任务，在注意力机制下不可能。

解决路径：小模型分工承担意图分析和状态管理，在每一轮为剧情生成模型装载当前场景的元信息（大纲、时间、地点、登场人物性格及关系、环境），卸载已离场的无关信息。剧情模型只做当下场景的生成，状态模型管理全局状态演进。明确告诉多个模型各自做什么，比让一个模型在多状态间反复纠结更合理。

核心问题

402 错误 — 三批根因各不相同。① assistant 永不被跳过，空填 "(thinking)"；② `_parse_owui_assistant()` 从 HTML 提取 thinking 重建块，显式指定 httpx 代理；③ drivers.py 显式传 `auth_token=api_key` 覆盖环境变量，防双头认证冲突。

DeepSeek 协议差异 — Anthropic 和 OpenAI 协议间需要翻译层，额外增加延迟和 token 占用。工具调用在 OpenAI 协议下更方便。

History: 格式 — 来自 OWUI Default Function Calling 的工具选择模板。open-terminal 扩展随机触发工具选择，把历史压缩为一行文本，不传递任何历史对话。修复：OWUI→设置→模型→Function Calling→Native/关闭。

OWUI 不回传 thinking — OWUI 数据库存了 thinking 但发给 API 时按连接类型选择性丢弃。OpenAI 兼容协议下全部丢弃。这是 UA Store、Side Store 等补丁的根本原因。
